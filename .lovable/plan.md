# خطة بناء بوابة إدارة وصيانة أصول جمعية الجبيل

## نظرة عامة
تطبيق ويب عربي (RTL) بأسلوب Liquid Glass، مستوحى من هوية جمعية الجبيل، لإدارة بلاغات صيانة المباني وتسعيرها ومتابعة ميزانياتها. مبني على TanStack Start + Lovable Cloud (قاعدة بيانات + مصادقة).

## المرحلة 1 — الأساس والتصميم
- تفعيل **Lovable Cloud** للمصادقة وقاعدة البيانات.
- بناء نظام التصميم في `src/styles.css`:
  - ألوان: أخضر تيركواز `#008375` (Primary)، كحلي `#0F2C59` (Secondary)، ذهبي `#D4AF37` (Accent).
  - توكنز زجاجية: `--glass-bg`, `--glass-border`, `--glass-blur`, ظلال ناعمة، تدرجات.
  - خط Readex Pro عربي عبر `<link>` في `__root.tsx`، اتجاه RTL على `<html>`.
- مكوّنات زجاجية قابلة لإعادة الاستخدام: `GlassCard`, `GlassButton`, `GlassStat`, `StatusPill`, `BudgetBar`.
- شعار وترويسة (Header) زجاجية علوية مع اسم الجمعية.

## المرحلة 2 — قاعدة البيانات والصلاحيات
جداول (public schema مع GRANTs وRLS):
- `profiles` — بيانات المستخدم + `assigned_building_id`.
- `user_roles` — enum: `admin`, `facility_manager`, `technician` + دالة `has_role` (Security Definer).
- `buildings` — المباني الخمسة (إدارة، مدرسة، سكنية، مستودع، مكاتب فرعية) + الميزانية السنوية.
- `facilities` — المرافق داخل/خارج كل مبنى (تصنيف: interior/exterior، نوع).
- `price_catalog` — كتالوج التسعير القياسي.
- `maintenance_requests` — البلاغات: مبنى، مرفق، وصف، حالة (pending/approved/in_progress/completed)، تكلفة تقديرية، تكلفة فعلية، فني معيّن، أولوية.
- `request_updates` — سجل التحديثات.

RLS:
- Facility Manager: يرى بلاغات مبناه فقط.
- Admin: كل شيء.
- Technician: البلاغات المعيّنة له.

Seed: 5 مباني، مرافق نموذجية، كتالوج أسعار (تكييف 150، مصعد 500، حدائق 300…).

## المرحلة 3 — المصادقة
- صفحة `/auth` (تسجيل دخول عبر البريد + كلمة السر).
- Layout محمي `_authenticated` (الـ integration-managed).
- إعادة توجيه حسب الدور بعد الدخول.

## المرحلة 4 — لوحات حسب الدور

### مسؤول المنشأة (`/dashboard`)
- بطاقة المبنى المخصص + شريط الميزانية الزجاجي (سنوي / مصروف / متبقي).
- زر "بلاغ جديد" → نموذج: اختيار المرفق → وصف → حساب تلقائي للتكلفة التقديرية من `price_catalog`.
- قائمة بلاغاته بحالاتها (Pills ملونة).

### مدير الصيانة (`/admin`)
- شبكة زجاجية بكل المباني مع مؤشرات الميزانية.
- جدول جميع البلاغات مع فلترة (مبنى/حالة/أولوية).
- إجراءات: اعتماد التسعير، تعديل التكلفة، تعيين فني.
- تقرير ميزانية إجمالي.

### الفني (`/technician`)
- قائمة مهامه، زر "بدء" و"إكمال" مع إدخال التكلفة الفعلية.

## المرحلة 5 — الصفحات المشتركة
- Landing `/` بواجهة زجاجية فخمة + CTA للدخول.
- تنقّل جانبي/علوي زجاجي تفاعلي.
- Toasts للإشعارات.

## التفاصيل التقنية
- TanStack Router (file-based) + TanStack Query.
- Server functions محمية بـ `requireSupabaseAuth` للعمليات الحساسة.
- ألوان عبر design tokens فقط (لا ألوان مباشرة في className).
- تأثيرات: `backdrop-filter: blur()`، حدود مضيئة، hover animations خفيفة.
- خط Cairo/Readex Pro، RTL كامل.

## الحدود (Scope)
- لن يشمل: رفع صور/مرفقات للبلاغات، إشعارات بريد، تكامل دفع، تطبيق موبايل. يمكن إضافتها لاحقاً.
- الشعار: سيُستخدم placeholder نصي/رمزي بألوان الهوية (لا يمكن سحب الشعار الرسمي تلقائياً؛ يمكنك رفع ملف الشعار لاحقاً لاستبداله).
